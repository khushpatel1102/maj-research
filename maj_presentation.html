MAJ

Memory Assisted Judge

Khush Patel | December 2024

What is MAJ?

An LLM-based judge that learns from past evaluations

Stateless Judge

  • Evaluates each case independently
  • No memory of past decisions
  • Inconsistent across similar cases

MAJ (Memory)

  • Stores past evaluations in Neo4j
  • Retrieves relevant examples
  • Learns from feedback

What Was Built

  • Core Pipeline - Extract nodes (Policy, Attempt, Issue, Fix) and store in Neo4j
  • Vector Embeddings - Similarity search for relevant memories
  • Three-Type Retrieval - Policies, contrastive examples, semantic patterns
  • Semantic Classification - Auto-categorize issues into high-level types

Type 1: Policies

Query: task embedding similarity

Returns: similar requirements/criteria

Type 2: Contrastive

Query: code/output embedding similarity

Returns: TOP 3 pass + TOP 3 fail examples

Type 3: Semantic Patterns

Query: issue embedding similarity

Returns: recurring issue categories (e.g., "SQL Injection" - 5 occurrences)

All three run in parallel and combine into one memory context

Fixes Applied

Memory was initially -10%, now neutral (0%)

  • Higher similarity thresholds - Only show highly relevant examples
  • Balanced pos/neg examples - Prevent rejection bias from too many failures
  • Anti-over-generalization prompts - "Judge THIS case on its own merits"

Limitations

  • Memory doesn't help with gpt-4o-mini (neutral effect)
  • Structured output overhead affects accuracy comparison
  • Cold start problem - need initial examples to be useful
  • Not tested on full dataset (100 samples vs 3,484)

Next Steps
